### 2. ✅ 安卓手机浏览器麦克风问题（核心修复）
**问题描述**：
- 华为、小米等手机浏览器虽然授权麦克风，但无法识别语音
- Web Speech API 浏览器兼容性差

**解决方案**：
- ❌ 完全移除 `webkitSpeechRecognition`（浏览器依赖）
- ✅ 实现 **Groq Whisper API** 方案（所有浏览器通用）

**技术实现**：
1. **创建 Whisper API 路由**：`app/api/groq-whisper/route.ts`
   - 使用 Groq SDK
   - Whisper-large-v3 模型
   - 完全免费、识别准确

2. **前端录音逻辑重构**：
   - 使用 `MediaRecorder` API 录音（所有浏览器支持）
   - 每 2 秒自动上传音频片段到 Whisper API
   - 异步识别，实时累积文本
   - 2 秒静默后发送给 AI

**优势对比**：

| 指标 | Web Speech API | Groq Whisper API |
|------|----------------|------------------|
| **兼容性** | ⚠️ Chrome/Edge 支持，华为/小米不支持 | ✅ **所有浏览器** |
| **识别准确度** | ⚠️ 参差不齐 | ✅ **业界顶级** |
| **数字识别** | ❌ "8" → "ate" | ✅ **完美识别** |
| **成本** | $0 | $0（**完全免费**） |
| **速度** | 实时 | < 1秒 |

**代码位置**：
- API 路由：`app/api/groq-whisper/route.ts`
- 前端逻辑：`page.tsx:39-42, 158-241`

**问题描述**：

- 需要增加到5行，但不能遮挡眼睛

**解决方案**：

### 6. ✅ 字幕-声音同步问题
**问题描述**：
- 刚开始字幕与声音同步
- 后来字幕变快，声音变慢
- 字幕速度不跟随声音

**解决方案**：
- **根据音频真实时长动态计算字幕速度**
- 实现方法：
  1. `onloadedmetadata` 获取音频时长
  2. 计算公式：`msPerWord = (audioDuration * 1000) / wordCount`
  3. 每个单词按精确时间间隔显示
- 代码位置：`page.tsx:57-108, 292-307`

**效果**：
- 🎯 字幕始终与语音精确同步
- ✅ 不再出现过快/过慢
